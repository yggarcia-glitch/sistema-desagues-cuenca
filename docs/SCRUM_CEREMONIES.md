# 🏃 Ceremonias SCRUM — Sistema de Desagüesobstruidos

## 1. Sprint Planning (Planificación del Sprint)
- **Duración:** 2 horas máximo por sprint de 2 semanas
- **Participantes:** Todo el equipo SCRUM
- **Objetivo:** Seleccionar historias del backlog y definir el Sprint Goal
- **Artefacto resultante:** Sprint Backlog

## 2. Daily Scrum (Reunión Diaria)
- **Duración:** 15 minutos máximo
- **Frecuencia:** Cada día del Sprint
- **Preguntas guía:**
  1. ¿Qué hice ayer para avanzar en el Sprint Goal?
  2. ¿Qué haré hoy?
  3. ¿Hay algún impedimento que me bloquea?

## 3. Sprint Review (Revisión del Sprint)
- **Duración:** 1 hora por sprint de 2 semanas
- **Participantes:** Equipo SCRUM + Stakeholders (ETAPA EP, docente)
- **Objetivo:** Demostrar el incremento funcional del sistema
- **Resultado:** Feedback para actualizar el Product Backlog

## 4. Sprint Retrospective (Retrospectiva)
- **Duración:** 45 minutos
- **Participantes:** Solo el equipo SCRUM
- **Preguntas guía:**
  1. ¿Qué salió bien en este Sprint?
  2. ¿Qué debemos mejorar?
  3. ¿Qué acciones concretas tomamos para el siguiente Sprint?

---

## 📊 Definition of Done (DoD)
Una historia de usuario se considera **DONE** cuando:
- [ ] El código está escrito y funcional
- [ ] Las pruebas unitarias pasan exitosamente
- [ ] La funcionalidad fue revisada por el Product Owner
- [ ] El código fue subido al repositorio (rama `main` o `develop`)
- [ ] La documentación fue actualizada
